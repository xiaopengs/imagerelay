服务器初始化脚本

# === 一键初始化（Ubuntu 22.04）=== sudo apt update && sudo apt upgrade -y # 安装 Docker curl -fsSL https://get.docker.com | sudo sh sudo usermod -aG docker $USER # 安装 Docker Compose（自动获取最新版本） COMPOSE=$(curl -s https://api.github.com/repos/docker/compose/releases/latest \ | grep '"tag_name"' | cut -d'"' -f4) sudo curl -L "https://github.com/docker/compose/releases/download/${COMPOSE}/docker-compose-$(uname -s)-$(uname -m)" \ -o /usr/local/bin/docker-compose && sudo chmod +x /usr/local/bin/docker-compose # 开放防火墙端口 sudo ufw allow 22/tcp && sudo ufw allow 80/tcp && sudo ufw allow 443/tcp && sudo ufw --force enable # 验证版本 docker --version # → ≥ 20.10 docker-compose --version # → ≥ 2.0

③ 部署步骤

7 步完成生产部署

1

克隆项目

git clone https://github.com/xiaopengs/imagerelay.git && cd imagerelay
2

构建前端

cd frontend && npm install && npm run build && cd .. # 复制构建产物 mkdir -p infra/frontend-dist && cp -r frontend/dist/* infra/frontend-dist/
3

申请 SSL 证书

sudo apt install certbot python3-certbot-nginx -y # 域名需提前解析到服务器 IP sudo certbot --nginx -d your-domain.com # 复制证书 mkdir -p infra/ssl sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem infra/ssl/cert.pem sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem infra/ssl/key.pem sudo chmod 644 infra/ssl/*.pem

⚠️ 域名需提前解析到服务器 IP，建议 DNS 生效后再继续

4

修改 Nginx 配置

编辑 infra/nginx.conf，将 your-domain.com 替换为真实域名

server_name your-domain.com; # ← 替换为你的域名
5

启动服务

cd infra && docker-compose up -d && docker-compose ps
6

One API 管理后台配置

① 访问 http://your-ip:3000

② 登录：root / 123456

③ 立即修改默认密码！

④ 渠道管理 → 添加 OpenAI

Key + 模型 gpt-image-2, dall-e-3

⑤（可选）添加 Google Gemini 渠道

⑥ 设置自定义首页、定价页

🐛 故障排查

502
原因：One API 容器未启动
解决：docker-compose ps → docker-compose restart
生成失败
原因：API Key 无效或渠道禁用
解决：管理后台 → 渠道管理 → 检查 Key 和余额
上传报错
原因：Nginx 文件大小限制过小
解决：nginx.conf 中 client_max_body_size 20M;
CORS 错误
原因：One API 未配置允许来源
解决：管理后台 → 系统设置 → 允许来源填写你的域名
